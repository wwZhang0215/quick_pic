# QuickPic — 照片筛图工具

轻量、快速的照片筛选工具，支持 JPG / RAW 配对管理，键盘驱动工作流，标记结果持久化。

支持平台：**Windows · macOS**

---

### 区域说明

| 区域 | 说明 |
|------|------|
| **照片显示区** | 占据左侧大部分窗口，显示当前照片。已标记照片右上角出现彩色角标（绿色 = KEEP，蓝色 = 文件夹键） |
| **统计面板** | 实时显示总张数、已标记总数，以及每个文件夹键的标记张数 |
| **EXIF 面板** | 显示当前照片的拍摄参数，切换照片时后台异步加载，不阻塞翻页 |
| **文件夹键 1–9** | 每行显示 `[键号] 文件夹名  (已标记张数)`，点击按钮可绑定或修改 |
| **默认保留文件夹** | 标记为 KEEP 但未绑定数字键时的默认移动目标，点击设置，× 清除 |
| **状态栏** | 底部一行：当前位置（第 N / 总数）、当前文件名、标记状态 |

## 安装

### 前提条件

- Python 3.11+

### 步骤

```bash
# 克隆仓库
git clone <仓库地址>
cd quick_pic

# 创建并激活虚拟环境
python -m venv .venv
source .venv/bin/activate        # macOS / Linux
# .venv\Scripts\activate.bat     # Windows CMD
# .venv\Scripts\Activate.ps1     # Windows PowerShell

# 安装依赖
pip install -e .
```

### 运行

```bash
python main.py
```

## 使用指南

### 1. 打开照片文件夹

菜单 **文件 → 打开文件夹…**，或按 `Ctrl+O`。

可以连续添加多个文件夹（弹窗询问是否继续添加），程序会合并所有照片并按**文件名**排序。

同名的 JPG 和 RAW 文件（如 `IMG_001.JPG` + `IMG_001.ARW`）自动配对，显示 JPG 以提升速度。

### 2. 浏览照片

| 按键 | 动作 |
|------|------|
| `←` | 上一张 |
| `→` | 下一张 |

### 3. 标记照片

| 按键 | 动作 |
|------|------|
| `K` 或 `Space` | 标记为**保留**（KEEP，绿色角标） |
| `1` – `9` | 标记移动到对应绑定文件夹（蓝色角标显示键号） |
| `U` 或 `Delete` | 取消标记 |

标记结果**实时写入**本地数据库（`~/.quickpic/data.db`），重启后自动恢复。

### 4. 绑定文件夹到数字键

在右侧面板点击 `[1]` – `[9]` 中的任意按钮选择目标文件夹。

也可以在浏览时按下未绑定的数字键，程序会弹窗提示立即绑定。

### 5. 执行移动

按 `M` 或菜单 **文件 → 移动已标记照片…**（`Ctrl+Shift+M`）：

1. 显示移动预览（各目标文件夹的文件数量）
2. 若有 KEEP 标记但未绑定文件夹的照片，弹窗要求指定目标路径
3. 确认后后台移动，进度实时显示
4. JPG 和 RAW 自动分开存放：

```
目标文件夹/
├── JPG/
│   └── IMG_001.JPG
└── RAW/
    └── IMG_001.ARW
```

移动完成后，已移动的照片从当前会话中清除。

### 6. 会话恢复

下次启动时，程序询问是否恢复上次的文件夹和浏览位置，选择"是"从上次位置继续。

## 快捷键一览

| 按键 | 动作 |
|------|------|
| `←` / `→` | 上一张 / 下一张 |
| `K` / `Space` | 标记保留 (KEEP) |
| `1` – `9` | 标记到对应文件夹 |
| `U` / `Delete` | 取消标记 |
| `M` | 执行移动 |
| `Ctrl+O` | 打开文件夹 |
| `Ctrl+Shift+M` | 执行移动（菜单） |
| `Ctrl+Q` | 退出 |

## 打包为可执行文件

打包必须在**目标平台**上执行，不支持跨平台交叉编译。

```bash
pip install pyinstaller
pyinstaller quick_pic.spec
```

- **Windows**：产物在 `dist\QuickPic\QuickPic.exe`，压缩整个 `dist\QuickPic\` 分发
- **macOS**：产物在 `dist/QuickPic.app`

GitHub Actions 在推送 tag 时自动在 Windows 和 macOS 分别构建，配置见 `.github/workflows/build.yml`。

```bash
git tag v0.1.0
git push origin v0.1.0
```
